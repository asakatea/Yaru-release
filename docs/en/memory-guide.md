# Memory Guide

## Audience and scope
The Memory module is where Yaru runs spaced repetition. Use it to manage decks, run reviews, and track long-term retention.

## Entry paths
- Main nav -> `Memory`
- Related paths from notes/editor flashcard actions

## Core capabilities
- Deck list and deck detail management
- FSRS-based review scheduling
- Review session ratings (`Again/Hard/Good/Easy`)
- Card creation from notes and editor blocks
- Import/export workflows (including compatible deck formats)

## Typical workflow
1. Create or open a deck.
2. Add cards from notes, editor, or manual input.
3. Start a review session and rate each card.
4. Check due count and daily progress.
5. Iterate with small daily sessions instead of large bursts.

## Practical tips
- Keep cards atomic: one fact per card.
- Use consistent wording for front/back fields.
- Review daily, even for short sessions.

## Troubleshooting
- No due cards: check deck filters and time settings.
- Unexpected schedule gaps: verify device time/timezone.
- Imported cards look wrong: validate source format and encoding.
