# 扩展模块指南

## 模块定位与适用人群
扩展模块是“知识捕获与自动化处理”中心，适合需要跨平台收集素材、订阅信息流、自动触发任务的用户。

## 入口路径
- 扩展首页：`/extensions`
- 媒体历史：`/extensions/media/history`
- 浮窗设置入口：`/extensions/floating`

## 功能总览
| 功能 | 说明 | 状态 |
|---|---|---|
| Captures | 展示捕获记录、同步、筛选、删除 | 已可用 |
| Feeds（RSS/Readwise） | 订阅源管理与条目浏览 | 已可用 |
| Media Synthesizer | 媒体摘要与历史记录 | 已可用 |
| Automation | Incoming/Outgoing Webhook、API Token | 已可用 |
| 浮窗设置入口 | 路由与设置按钮存在，功能仍在建设中 | 开发中 |
| Captures 一键转卡片/笔记 | 当前为 pending 提示，未接入目标流程 | 开发中 |
| 自动化日志查看 | 部分日志查看流程仍有 TODO | 实验中 |

## 典型任务流程
1. 打开 Captures 同步最新捕获内容。
2. 在 Feeds 添加 RSS 源并按需阅读。
3. 在 Media 处理视频/媒体摘要并查看历史。
4. 在 Automation 创建 Webhook 与 API Token。
5. 将内容分发到笔记、练习或记忆场景。

## 高级用法与效率技巧
- Captures 先打标签再批处理，后续检索成本更低。
- Automation 建议按“来源系统”拆分 Token 与 Webhook，便于审计。
- Feeds 与笔记联动可构建稳定的信息输入管道。

## 权限/角色/前置条件
- 需要登录账号。
- Automation 依赖后端接口与 token 权限范围。

## 常见问题与排错
- 同步无更新：先手动触发 Sync，再检查网络。
- Webhook 回调失败：检查目标地址、签名与事件类型配置。
- 浮窗页面为空：该功能目前处于开发中阶段。

## 相关模块联动
- 扩展 -> 笔记：捕获内容沉淀为结构化笔记。
- 扩展 -> 记忆：将高价值信息转卡片。
- 扩展 -> 协作：把内容投递到论坛或团队讨论流。
