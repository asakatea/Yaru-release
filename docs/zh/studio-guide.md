# 创作工作室指南

## 模块定位与适用人群
Studio 是内容生产与治理中心，面向创作者、审核人员和管理员，支持题目、书库、Wiki、课程等资源的全流程管理。

## 入口路径
- Studio 首页：`/studio`
- Problems：`/studio/problems`
- Library：`/studio/library`
- Wiki：`/studio/wiki`
- Courses：`/studio/courses`
- Review：`/studio/review`
- Users：`/studio/users`

## 功能总览
| 功能 | 说明 | 状态 |
|---|---|---|
| Studio 首页 | 模块卡片、快捷创建、导入入口 | 已可用 |
| 题目与资源管理 | 题目资源、书籍资源、Wiki 内容编辑 | 已可用 |
| 课程管理 | 课程列表、创建、编辑、状态管理 | 已可用 |
| 审核中心 | 待审核资源处理 | 已可用 |
| 用户管理 | 管理用户信息与角色（管理员） | 已可用 |
| 仪表盘部分组件 | 个别卡片仍为开发中结构 | 实验中 |

## 典型任务流程
1. 进入 `/studio` 选择创建类型（题目/内容/导入）。
2. 在对应模块完成草稿编辑。
3. 提交审核并在 `/studio/review` 跟进状态。
4. 发布后同步到学习、书库或题库消费侧。
5. 通过 `/studio/users` 维护协作团队角色。

## 高级用法与效率技巧
- 按内容类型拆分工作流，减少跨模块切换成本。
- 先建模板再批量生产，提升内容一致性。
- 审核阶段建议使用统一标准（结构、准确性、可练习性）。

## 权限/角色/前置条件
| 角色 | 访问范围 |
|---|---|
| donor/contributor | 可进入 Studio 主体功能 |
| admin/owner/content_admin | 额外可访问审核与用户管理 |
| 普通学习用户 | 默认不可进入 Studio |

## 常见问题与排错
- 无法访问 `/studio`：通常是角色不在允许列表中。
- 用户页打不开：需要管理员级权限。
- 审核数据为空：可能当前无待审核资源或筛选条件不匹配。

## 相关模块联动
- Studio -> 学习：课程发布后进入学习端。
- Studio -> 练习：题目资源可直接进入练习系统。
- Studio -> 书库/Wiki：内容资产统一管理与消费。
