## 典型任务流程
1. 当你第一次使用 Yaru 时，先完成 `getting-started` 的首次任务链。
2. 当你希望系统学习时，按“学习 -> 练习 -> 记忆”顺序推进。
3. 当你做阅读型学习时，按“书库 -> 笔记 -> 记忆”沉淀关键知识。
4. 当你遇到故障时，直接跳转 `troubleshooting-guide` 按场景排查。

## 高级用法与效率技巧
- 当你每天学习时间有限时，优先完成“1 次课程 + 1 次练习 + 1 次复习”的最小闭环。
- 先用默认设置跑通主流程，再进入高级设置调同步、主题和通知策略。
- 以“问题驱动”阅读文档：先定位场景，再看对应模块，避免一次性阅读过多内容。

> [提示] 如果你不确定先看哪篇文档，默认从 `入门指南 -> 学习指南 -> 记忆指南` 开始。

## 权限/角色/前置条件
- 普通用户主线功能需要登录账号。
- Studio 属于高权限创作区，默认普通学习用户不可访问。
- 团队协作中的成员管理与角色调整受团队权限控制。

## 常见问题与排错
- 我不知道先看哪篇：先看 `入门指南`，再按本手册的四层结构走。
- 文档太多看不完：按“当前任务”选择对应手册，不必全读。
- 某些页面显示开发中：属于已知迭代范围，不影响主学习闭环。
